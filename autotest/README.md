# autotest — линейка «иишности» (dev-тулинг)

Детерминированный скрипт, который ловит **очевидные** ИИ-маркеры по regex-каталогу и
ставит грубую оценку «AI-likeness». Это **измеритель прогресса для разработки скилла**,
а НЕ его рантайм-детектор: сам скилл остаётся суждением модели по принципам (см.
`../SKILL.md`). Каталог — грубые эвристики; severity отражает риск ложного срабатывания.

## Запуск

```
node autotest/autotest.mjs score <file>     # оценка одного файла + список маркеров
node autotest/autotest.mjs scan  <dir>      # таблица по всем .txt/.md в папке
node autotest/autotest.mjs gate  <file|dir> # exit 1, если остались hard/strong маркеры
node autotest/autotest.mjs self             # проверка дискриминации на корпусе
```

## Важно

- **`gate`/hard/strong применять только к ВЫВОДУ чистки**, не к диагнозу живого текста:
  живой человек тоже пишет «важно отметить» и ставит тире. Многие паттерны структурные и
  срабатывают на легитимной формальной прозе (наука/пресс) — суди по со-встречаемости и
  регистру, не по одиночному совпадению.
- `self` ожидает папку `../corpus` с образцами живого текста; в репозиторий она не входит
  (приватный/тяжёлый скрапленный контент), так что `self` без неё покажет пустой человеческий
  список. `score`/`scan`/`gate` работают автономно.
- `patterns.json` растёт из каждого раунда калибровки. `severity`: `hard` (почти всегда ИИ) /
  `strong` (высокое подозрение) / `weak` (человек тоже так пишет, только в связке).
