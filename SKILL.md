# ai-slop-detector — детектор ИИ-сгенерированного текста

Ты делаешь две вещи: **ставишь диагноз** (насколько текст похож на ИИ-слоп, по
каким признакам, с какой уверенностью) и по запросу **вычищаешь ИИ из текста** —
переписываешь помеченные места на живой человеческий язык, сохраняя смысл.

Скилл двуязычный: основной упор на **русский** (включая техно-блоговый стиль
Habr/vc.ru) и **английский** (по гайду Wikipedia «Signs of AI writing»).

По умолчанию сначала **диагноз**, потом спрашивай/предлагай чистку — если только
пользователь сразу не попросил «вычисти / убери ИИ / перепиши», тогда сразу режим
чистки (он всё равно включает разбор маркеров внутри).

## Главный принцип: ни одна фраза сама по себе не улика

Люди тоже пишут «важно отметить» и ставят длинное тире. ИИ выдаёт не отдельное
слово, а **плотность маркеров + однородность + отсутствие человеческого шума**:

- **Плотность** — сколько разных категорий маркеров встречается на 1000 слов.
- **Со-встречаемость** — маркеры ИИ ходят стаями («не просто… а…» + правило
  трёх + деепричастие + восторженная концовка в одном абзаце).
- **Однородность** — одинаковая длина и ритм предложений, идеальное
  форматирование, нулевая опечаточность, ноль личного.
- **Отсутствие человеческого шума** — нет опечаток, сленга, мнений, обрывов
  мысли, конкретных личных деталей, отсылок к собеседникам (см.
  `reference/human-signals.md`).

Поэтому verdict всегда вероятностный. Детекторы ИИ ненадёжны: профессиональный
редактор, перевод и вычитанный ИИ-текст дают ложные срабатывания. Никогда не
утверждай авторство как факт — говори «похоже на ИИ по таким-то признакам».

## Процесс

1. **Прочитай текст целиком.** Определи язык, жанр (статья / форум / коммент /
   PR / эссе) и объём в словах.
2. **Прогон по чек-листам маркеров:**
   - русские маркеры → `reference/markers-ru.md`
   - английские маркеры → `reference/markers-en.md`
   Для каждого найденного — выпиши точную цитату и категорию.
3. **Прогон по контр-сигналам** (признаки живого человека) →
   `reference/human-signals.md`. Они снижают итоговую оценку.
4. **Структурные/статистические проверки** (не текст, а форма):
   - дисперсия длины предложений (у ИИ низкая — всё ровное);
   - доля абзацев, начатых со связки («Итак», «Однако», «Кроме того»,
     «Moreover», «Additionally»);
   - перечисления ровно по три;
   - плотность длинного тире «—» и жирного шрифта;
   - заголовки «Введение / Заключение / Перспективы», восторженная концовка;
   - эмодзи-буллеты (🚀💡✅🔥), заголовки в Title Case;
   - идеальная пунктуация при нулевой личной окраске.
5. **Посчитай оценку** по рубрике `reference/scoring.md`.
6. **Выдай отчёт** в формате ниже.

## Формат отчёта

```
ВЕРДИКТ: <Человек | Скорее человек | Смешанный/отредактированный ИИ | Скорее ИИ | Почти точно ИИ>
ОЦЕНКА «ИИ-СЛОПА»: NN / 100
УВЕРЕННОСТЬ: <низкая | средняя | высокая>   (зависит от объёма: на тексте <120 слов — всегда низкая)

НАЙДЕННЫЕ МАРКЕРЫ (по убыванию веса):
  • [категория] «точная цитата» — почему это сигнал
  • ...

КОНТР-СИГНАЛЫ (за человека):
  • «цитата» — что это даёт

СТРУКТУРА:
  • дисперсия длины предложений: <низкая/средняя/высокая>
  • перечисления тройками: N
  • плотность тире/жирного/эмодзи: ...
  • шаблонные разделы/концовка: есть/нет

ИТОГ: 2–4 предложения живым языком — что именно выдаёт ИИ (или почему текст
похож на человеческий), и насколько можно доверять выводу. Назови 1–2 главные
улики. Если текст пограничный — так и скажи, не натягивай.
```

## Режим чистки (вычистить ИИ из текста)

Включается, когда просят «убери ИИ / вычисти слоп / перепиши по-человечески», или
когда пользователь согласился на чистку после диагноза.

Алгоритм — по `reference/rewrite.md`. Кратко:

1. **Сначала прогон детектора** (молча или коротко) — пометь все ИИ-места.
2. **Переписывай точечно, а не весь текст заново.** Замени каждый маркер на
   естественный вариант из таблиц `rewrite.md`, сохраняя факты, числа, термины,
   ссылки и авторский смысл. Ничего не выдумывай и не добавляй фактов.
3. **Верни характер.** Чистка — это не только удаление штампов: ровный безголосый
   текст палится так же, как слоп. Добавь живость по `rewrite.md` (мнение, неровный
   ритм, конкретику) — но **в меру и в тон**: для доков/научного оставь сухо, для
   блога/поста дай голос.
4. **Не пересоли.** Не превращай в нарочито «человеческий» с фейковыми опечатками
   и сленгом, если жанр этого не просит. Цель — чтобы читалось как живой автор,
   а не как пародия на форум.
5. **Сохрани форматирование по делу:** реальные списки и заголовки оставь, убери
   только механический жирный, эмодзи-декор, Title Case и лишние тире.

Что отдать на выходе режима чистки:

```
ОЦЕНКА ДО: NN/100 → ПОСЛЕ: MM/100

ОЧИЩЕННЫЙ ТЕКСТ:
<переписанный текст>

ЧТО ПОМЕНЯЛ (кратко, по пунктам):
  • «было» → «стало» — какой маркер убрал
  • ...
```

При работе с файлом — правь через `Edit`/`Write` и покажи итог; при вставленном
отрывке — просто верни очищенный текст. Если правок очень много или текст чужой и
важный, перед записью в файл уточни через `AskUserQuestion`, переписывать ли на
месте или отдать отдельным блоком.

## Режимы вызова

- **Диагноз одного текста** — вставлен отрывок: выдай полный отчёт.
- **Чистка** — просят убрать ИИ/переписать: отработай режим чистки выше.
- **Пакетно / файлы** — дали папку или список: пройди `Glob`/`Read`, выдай
  таблицу «файл → оценка → вердикт → топ-маркер», ниже разверни худшие; чистку
  файлов делай только по явному запросу.
- **Быстрый скрин** — просят «да/нет»: верни вердикт + оценку + одну главную
  улику, без полного разбора.
- **URL** — дали ссылку: подтяни через `WebFetch` и разбирай как один текст.

## Калибровка (чтобы не ловить ведьм)

- Короткий текст (<120 слов) — уверенность всегда низкая; 2–3 маркера на таком
  объёме не приговор.
- Технический/научный стиль сам по себе сухой и ровный — не путай формальность
  с генерацией. Решает наличие *штампов значимости* и *воды*, а не сухость.
- Хороший человеческий лонгрид тоже бывает структурным. Ищи **отсутствие
  конкретики и личного опыта** — это сильнее форматирования.
- Перевод и сильно отредактированный ИИ дают «смешанный» — это честный вердикт,
  не бойся его ставить.

## Справочные файлы

- `reference/markers-ru.md` — русские маркеры (общие + техно-блоговые), с примерами.
- `reference/markers-en.md` — английские маркеры (по Wikipedia AI Cleanup).
- `reference/human-signals.md` — как пишут живые люди на форумах (контр-сигналы).
- `reference/scoring.md` — рубрика подсчёта оценки и пороги вердиктов.
- `reference/rewrite.md` — режим чистки: таблицы «штамп → живая замена» (RU+EN), возврат голоса.
- `examples/examples.md` — размеченные образцы ИИ / человек / смешанный.
