# Changelog

Все заметные изменения скилла. Формат — по убыванию версий.
All notable changes, newest first.

## 2.1.0
- **Анти-метроном.** Рубленый ровный ритм — главный остаточный тейл прозы, и сама чистка его порождает.
  Добавлено правило-паттерн «варьируй длину предложений» + самоверификация прогоняет burstiness СВОЕГО
  вывода (метроном помечен как самая частая ошибка чистки).
- **Примеры → общие паттерны.** Убраны прескриптивные «❌ не делай это → ✅ делай то» / «До → После»
  worked-примеры (rewrite.md Примеры 1–3, правила чистки художки); заменены на паттерны «тейл → стратегия».
  Так модель обобщает, а не подражает конкретным (часто рубленым) образцам. Сохранены: фразы-сигнатуры
  маркеров и калибровочные образцы `examples/examples.md` (другой назначение).

## 2.0.0
- **Художка/проза** (`reference/markers-fiction.md`): запах/холод-клише («Пахло X», кровь=медь/железо),
  фильтр-слова и «show don't tell», пафосные синонимы, вода, тройки, метроном, финал-бантик + правила чистки.
- **Короткие/платформенные форматы** (`reference/markers-shortform.md`): YouTube-заголовки/сценарии/комменты,
  LinkedIn/X, форум-комменты — со своими каркасами и регистрами.
- **Структурные/статистические сигналы** (`reference/structural-signals.md`): burstiness (разброс длины
  предложений), em-dash density, плотность связок-зачинов, copula-dodge, повтор зачинов, лексическое
  разнообразие — переживают замену синонимов. Плюс **форензик хард-теллы** (`oaicite`, `contentReference`,
  `utm_source=chatgpt`, `:::`) как мгновенный максимум.
- **Поведенческие фишки** (перенято у humanizer-конкурентов): самоверификация чистки (detector-in-the-loop —
  не отдавать текст, который валит собственный детектор), per-sentence heat-map, уровни чистки (light/standard/deep).
- Эвристика: burstiness, hard-tells, copula-dodge, художка/YT/LinkedIn-маркеры; +5 eval-кейсов (eval 22/22).
- Источники: 4 research-сабагента (Wikipedia AI Cleanup, GPTZero, arXiv stylometry, Aboudjem/jpeggdev/
  harshaneel/humanizerai humanizer-скиллы, рус. разборы нейропрозы).

## 1.5.0
- Новая категория маркеров (RU + EN): **формат коротких сообщений** (чат / мессенджер / письмо) —
  двоеточия-анонсы («Основание простое:», «Коротко:»), двойные переносы между абзацами,
  шаблон «приветствие → список из трёх → вежливый CTA», длинное тире и идеальная типографика в чате.
- Регистр **«чат / мессенджер / DM»** добавлен в `rewrite.md` (шаг 0) с таблицей нормализации формата
  и примером (просьба о повышении). Контр-сигналы формы — в `human-signals.md`.
- Эвристика (`scripts/markers.mjs`) ловит двоеточия-анонсы и форму коротких сообщений; +2 чат-кейса в eval.

## 1.4.0
- CI: GitHub Actions (`.github/workflows/ci.yml`) гоняет `validate.mjs` + `eval/run.mjs` на каждый push/PR.
- Добавлен `validate.mjs` — самопроверка SKILL.md по спецификации Agent Skills (без Python; `node validate.mjs`).
- Добавлен `scripts/scan.mjs` — пакетный скан-триаж: считает плотность маркеров по файлам/папке.
- Добавлен `scripts/markers.mjs` — общий набор regex-маркеров и эвристика band'а для инструментов.
- Добавлен eval-харнесс: `eval/cases.json` (размеченные сниппеты) + `eval/run.mjs` (регрессия по полосам вердикта).
- Расширены маркеры (RU + EN): «стоит упомянуть», «нельзя обойти стороной», "when it comes to", "in conclusion", "let's dive/explore".
- Новые примеры-эталоны: перевод → «смешанный», очень короткий текст → низкая уверенность.
- `.gitattributes` (eol=lf) — убирает LF/CRLF-шум.
