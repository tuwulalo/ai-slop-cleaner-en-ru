# Режим чистки: вычистить ИИ из текста

Цель — убрать машинные штампы и вернуть тексту живого автора, **сохранив смысл,
факты, числа, термины и ссылки**. Не добавляй информации, которой не было.

Goal: strip AI-isms and restore a human voice **without changing meaning, facts,
numbers, terms or citations**. Never invent content.

Порядок: (1) прогон детектора → пометить; (2) точечная замена по таблицам ниже;
(3) вернуть голос по жанру; (4) проверить, что смысл и факты на месте.

---

## Принцип замены

Для каждого маркера действуй так:

1. **Штамп значимости/воду — выкинуть, дать факт.** Не «играет ключевую роль», а
   что именно делает.
2. **Связку-конферанс — убрать или свести к делу.** «Давайте разберёмся, что под
   капотом» → сразу объяснение.
3. **Копулу вернуть.** «является/представляет собой/выступает в роли» → «это / —».
4. **Деепричастный хвост псевдоглубины — отрезать**, если смысл не меняется.
5. **Тройку — сжать** до одного-двух точных слов.
6. **Обтекаемую ссылку — конкретизировать или убрать.** «эксперты считают» → кто
   именно, либо «я думаю», либо вычеркнуть.

## Таблица замен — русский

| Было (ИИ) | Стало (живое) |
|-----------|----------------|
| важно отметить, что X | X (просто факт) |
| стоит отметить / следует учитывать | убрать; если нужно — «учти:» |
| является / представляет собой | это / — (тире) |
| играет ключевую роль в | нужен для / отвечает за |
| в современном мире / в наше время | убрать или назвать конкретный контекст |
| не секрет, что | убрать |
| это не просто X, а Y | X. И ещё Y (либо одно из двух) |
| не только X, но и Y | X и Y |
| давайте разберёмся / погрузимся | сразу к сути, без вступления |
| под капотом | внутри / в реализации / устроено так (и не повторять) |
| разложим по полочкам / простыми словами | убрать, объяснять сразу |
| и вот тут на сцену выходит X | тут используется X / дальше — X |
| открывает новые возможности | даёт возможность <какую конкретно> |
| …, тем самым обеспечивая / способствуя | отрезать хвост или «— это <эффект>» |
| эксперты считают / исследования показывают (без ссылки) | <кто/где> или «по-моему» или убрать |
| может стать / может быть полезным | станет / поможет <в чём>, либо убрать модальность |
| надеюсь, статья была полезной / спасибо за внимание | удалить (это обращение чат-бота) |
| Конечно! / Отличный вопрос! / Вы абсолютно правы | удалить |
| быстро, удобно и надёжно (тройка) | оставить одно точное: «быстро» или «надёжно» |
| Будущее выглядит многообещающим | конкретный план/факт: «дальше — <что>» |

| AI | Human |
|----|-------|
| it's not just X, it's Y | X. It also Y. |

| serves as / stands as a | is |

| robust, seamless, scalable (cluster) | keep one accurate word |

## Вернуть голос (по жанру, в меру)

Чистка без голоса = всё ещё мёртвый текст. Добавляй ровно столько, сколько уместно:

- **Мнение вместо нейтрального отчёта:** «по-моему, это перебор», «сработало, но
  осадочек остался». Не у каждого предложения — точечно.
- **Неровный ритм:** длинную мысль чередуй с короткой. «И всё. Сломалось.» —
  нормально.
- **Конкретика вместо обобщений:** если в исходнике были числа/имена/версии —
  сохрани их; не заменяй точное на «революционное».
- **Честная неуверенность по сути** (не шаблонный хедж): «не уверен, что это
  лучший путь, но у нас взлетело».

### Дозировка по жанрам

| Жанр | Сколько голоса |
|------|----------------|
| Док / API / научное | минимум: только убрать штампы, оставить сухо и точно |
| Блог / Habr-пост | средне: мнение + ритм, без фейкового сленга |
| Форум / коммент / соцсети | много: разговорно, можно сленг и обрывы — но не пародия |
| Маркетинг/лендинг | убрать восторг и тройки, оставить конкретную пользу |

---

## Чего НЕ делать при чистке

- Не добавлять факты, цифры, источники, которых не было в оригинале.
- Не выбрасывать настоящие данные ради «чистоты» — числа и термины священны.
- Не вставлять нарочитые опечатки и сленг там, где жанр их не терпит (это такой же
  палёный приём, как и слоп).
- Не переписывать весь текст заново, если хватает точечных замен — так теряется
  авторский смысл.
- Не трогать цитаты и код: внутри кавычек и блоков кода ничего не «чистим».

## Пример (RU, блог)

**До (76/100):**
> В современном мире кэш — это не просто хранилище, а целый механизм ускорения.
> Важно отметить, что под капотом используется LRU, тем самым обеспечивая
> вытеснение редких данных. Будущее кэширования выглядит многообещающим! 🚀

**После (12/100):**
> Кэш у нас не просто хранилище — он ещё и вытесняет редко используемые ключи.
> Внутри обычный LRU: когда место кончается, выкидывается самый давно не
> запрошенный ключ. Просто и предсказуемо — нам хватает.

**Что поменял:**
- «в современном мире», «важно отметить», «выглядит многообещающим», эмодзи — удалил;
- «является/целый механизм» → копула и факт;
- «тем самым обеспечивая» → прямое объяснение, что делает LRU;
- добавил живое «нам хватает» (блог-жанр) без выдумывания фактов.
